现在我需要你帮我设计一个 **审批系统**，要求如下：

## 1. 架构目标
- 系统要支持 **多种审批类型**（如财务审批、项目审批、合作者审批等），并且方便后续扩展新的审批类型，如果审批类型没有匹配，则默认为 **通用审批**。
- 系统要支持 **多种节点类型**，比如：开始节点、结束节点、条件节点、审核节点、抄送节点、子流程节点等。并且方便后续扩展新的节点类型。
- 审批流程要支持：
    - **会签**（所有人同意才通过）
    - **并签**（任意一人同意即可通过）
    - **子流程**（某节点可以挂载其他审批流程模版）
- 审批动作包括：**同意、驳回、抄送**。
- 架构要分层清晰：
    - **Controller 层**（接收请求）
    - **Service 层**（业务逻辑）
    - **审批引擎/上下文**（基于设计模式处理流程逻辑）
    - **Repository 层**（数据库持久化）
- 系统要支持 **回调**，比如：
    - **审批通过**：发送短信、站内信、邮件、回调
- 特别注意：审批流程不要一次性创建，而是动态创建。这方便回退节点
- 创建审批流程时，增加草稿状态（create），也可以直接提交 is_draft = true 创建草稿， is_draft = false 创建正式流程

## 2. 技术栈

- 使用 **设计模式** 提升扩展性，比如：
    - **策略模式**：处理不同审批类型（财务、项目…）
    - **责任链模式**：处理审批节点流转
    - **状态模式**：处理审批状态（待审批、已同意、已驳回…）
    - **工厂方法模式/抽象工厂模式**：动态创建不同审批流对象
    - **观察者模式**：处理通知短信、站内信、邮件、回调
- 场景匹配表

    | 需求               | 推荐模式      | 作用                |
    |------------------|-----------|-------------------|
    | 审批模版创建（节点配置）     | **建造者模式** | 按步骤构建模版           |
    | 节点支持子流程 / 并行     | **组合模式**  | 节点层级化管理           |
    | 业务触发创建审批流        | **工厂模式**  | 不同业务产出不同审批流       |
    | 审批流的状态流转         | **状态模式**  | 流程状态解耦            |
    | 同意 / 驳回 / 子流程等操作 | **策略模式**  | 扩展不同操作逻辑          |
    | 节点逐级流转           | **责任链模式** | 动态处理节点审批          |
